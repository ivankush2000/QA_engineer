# Проекты 📋
## Проект 1: Тестирование личного кабинета для учителей Skyeng 

📌**Описание проекта:**

Объект тестирования — модуль расписания и планирования уроков/событий личного календаря преподавателя на базе крупной образовательной онлайн-платформы (Skyeng.ru).
Модуль представляет собой интерактивный календарь-планировщик для учителей, позволяющий создавать регулярные уроки, добавлять личные события, настраивать длительность и цветовую индикацию занятий, а также интегрироваться с бэкендом расписания через специализированное API (https://api-teachers.skyeng.ru/v2/schedule/).

🎯**Цель проекта:**

Обеспечить надежность, логическую стабильность и корректность работы функционала планирования. Ключевая цель — верифицировать отсутствие конфликтов в расписании, корректность обработки временных зон и лимитов (как на фронтенде, так и на бэкенде через API-запросы), а также гарантировать бесшовный и интуитивно понятный пользовательский опыт для преподавателей.

[_**Ссылка на тест-план**_](https://1703h.yonote.ru/share/9552a3d4-9df3-4463-903d-fd414aa3556e)

[_**Ссылка на отчет о тестировании**_](https://1703h.yonote.ru/share/ee11c44b-826b-4dbb-8f69-6f9d497adb55)


🔍**Виды тестирования.**

В рамках проекта был выполнен комплекс тестовых мероприятий:

- Тестирование требований: анализ спецификации на ранних этапах, составление списка уточняющих вопросов (валидация длины полей, поддержка символов кириллицы/латиницы, лимиты дат и времени в прошлом/будущем).

- Дымовое тестирование: проведение быстрого прогона критического пути (создание простого события в календаре, базовый запрос к API) для подтверждения стабильности сборки.

- Функциональное тестирование: детальная ручная проверка логики создания, изменения уроков и личных событий через UI и API.

- Тестирование API: функциональное и негативное тестирование бэкенда (POST /v2/schedule/createPersonal) для проверки ограничений, которые не пропускает фронтенд (длительность более 24 часов, пустые поля, спецсимволы и пробелы).

- Регрессионное тестирование: повторные проверки для подтверждения отсутствия новых багов в существующем функционале календаря после внесения изменений.

- Приемочное тестирование: валидация разработанного функционала на соответствие бизнес-требованиям и критериям приемки.


🛠**Какие инструменты были применены:**

- Test IT - TMS-система для планирования тест-дизайна, ведения тест-планов и фиксации результатов прогона тест-ранов (Smoke, Acceptance, Regression).

- Postman - создание и выполнение коллекций API-тестов, верификация поведения эндпоинта создания личных событий преподавателей, валидация негативных сценариев бэкенда.

- Miro — декомпозиция интерфейса, построение ментальных карт (Mind Maps) логических связей календаря и сценариев взаимодействия.

- Microsoft Excel - проведение анализа требований, ведение функциональных и регрессионных чек-листов, фиксация вопросов к бизнес-логике.

- Google Chrome, Microsoft Edge - кроссбраузерное тестирование UI календаря, отладка сетевых запросов через Chrome DevTools.

## Проект 2: Тестирование онлайн-кинотеатра и информационного портала «Кинопоиск»

📌**Описание проекта:**

«Кинопоиск» — крупнейший в России онлайн-кинотеатр и мультимедийный информационный портал о киноиндустрии. Платформа предоставляет пользователям возможность удобного поиска фильмов и сериалов, просмотра трейлеров и медиаконтента, публикации рецензий, выставления оценок, а также получения персональных рекомендаций на основе индивидуальных зрительских предпочтений.

В рамках данного проекта тестировались ключевые функциональные модули веб-версии портала: глобальная поисковая строка (включая фильтрацию результатов по различным параметрам), форма авторизации пользователей, интерактивный медиаплеер и блок пользовательских рецензий. Особый фокус был сделан на интеграции фронтенда с бэкенд-сервисами посредством API-интерфейса.

🎯**Цель тестирования**

Главная цель тестирования — комплексная оценка качества, работоспособности и безопасности критического функционала веб-версии «Кинопоиск», а также верификация надежности API-интеграции.

В ходе проекта проверялось:

1. Поиск и фильтрация: корректность работы поисковой строки на главной странице и фильтров поиска.
2. Авторизация пользователей: надежность процесса авторизации, валидация полей ввода и защита от некорректных данных.
3. Воспроизведение контента: стабильность работы элементов управления интерактивного плеера.
4. Раздел «Рецензия зрителя»: возможность создания, редактирования, валидации и отправки рецензий пользователей.
5. API-интеграция: корректность обработки сервером входящих параметров.
6. Нефункциональные характеристики: адаптивность верстки, скорость загрузки страниц, доступность интерфейса при масштабировании и клавиатурной навигации, а также общее удобство использования.

🔍**Полный перечень применённых видов тестирования**

В проекте был реализован гибкий подход, сочетающий в себе функциональные и нефункциональные виды проверок:

1. Функциональное тестирование:
- Поиск и фильтрация: проверка точности поисковой выдачи по различным типам запросов и корректности применения сложных фильтров.
- Авторизация: проверка позитивных сценариев входа в систему и негативных кейсов.
- Пользовательский контент: тестирование логики создания и публикации рецензий.

2. Дымовое тестирование:
- Оперативная проверка ключевых функций после деплоя для подтверждения базовой работоспособности сборки.

3. Тестирование пользовательского интерфейса:
- Тестирование медиаплеера: проверка всех UI-компонентов плеера.
- Визуальный контроль корректности отображения текстовых блоков, кнопок и форм на главной странице и в карточках фильмов.

4. Тестирование API:
- Позитивные проверки: отправка валидных HTTP-запросов к бэкенду портала, верификация статус-кодов и структуры полученных JSON-ответов.
- Негативные проверки:
  - Отправка запросов с невозможными значениями параметра page.
  - Отправка запросов с id фильма/сериала, выходящим за пределы допустимого диапазона.

5. Нефункциональное тестирование:
- Тестирование совместимости и кроссбраузерности: проверка корректности рендеринга страниц, работы скриптов и адаптивности интерфейса в браузерах Google Chrome и Microsoft Edge при различных разрешениях экрана.
- Тестирование производительности: проверка и замер времени загрузки главной страницы сайта для контроля соответствия SLA по скорости работы интерфейса.
- Тестирование доступности: проверка читаемости и масштабируемости текста, а также доступности интерфейса для навигации с клавиатуры.
- Тестирование удобства использования: оценка эргономики интерфейса, простоты навигации по каталогу фильмов и интуитивности взаимодействия с плеером и формой рецензий.
